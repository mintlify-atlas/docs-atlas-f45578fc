## Overview

Cossistant provides Docker configurations for both development and production deployments. The setup includes:

- **API Service**: Containerized API server (Bun runtime)
- **Workers Service**: Containerized background workers (Bun runtime)
- **PostgreSQL**: Database with pgvector extension
- **Redis**: Caching and session storage

## Docker Compose Setup

The included `docker-compose.yml` provides PostgreSQL and Redis services for local development.

### Starting Infrastructure

```bash
# Start PostgreSQL and Redis
docker compose up -d

# View logs
docker compose logs -f

# Stop services
docker compose down

# Stop and remove volumes (WARNING: deletes data)
docker compose down -v
```

### Docker Compose Configuration

```yaml docker-compose.yml
services:
  postgres:
    image: pgvector/pgvector:pg17
    restart: unless-stopped
    ports:
      - "5432:5432"
    environment:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres
      POSTGRES_DB: cossistant
    volumes:
      - postgres_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U postgres"]
      interval: 10s
      timeout: 5s
      retries: 5

  redis:
    image: redis:7-alpine
    restart: unless-stopped
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 10s
      timeout: 5s
      retries: 5

volumes:
  postgres_data:
  redis_data:
```

## API Container

The API service runs the main HTTP server using Bun.

### Dockerfile.api

```dockerfile Dockerfile.api
FROM node:20-slim

WORKDIR /app

# Install Bun
RUN npm install -g bun

# Copy package files
COPY package.json bun.lock turbo.json ./
COPY apps/api/package.json ./apps/api/
COPY packages/typescript-config/package.json ./packages/typescript-config/
COPY packages/core/package.json ./packages/core/
COPY packages/transactional/package.json ./packages/transactional/
COPY packages/types/package.json ./packages/types/
COPY packages/jobs/package.json ./packages/jobs/
COPY packages/facehash/package.json ./packages/facehash/
COPY packages/tiny-markdown/package.json ./packages/tiny-markdown/
COPY packages/redis/package.json ./packages/redis/
COPY packages/location/package.json ./packages/location/
COPY tinybird/package.json ./tinybird/

# Install dependencies
RUN bun install

# Copy source files
COPY . .

# Build dependencies (transactional, types, location packages)
# but skip the API binary compilation
RUN bunx turbo run build --filter=@cossistant/api^...

# Railway will set the PORT environment variable
# EXPOSE is mainly for documentation, the actual port is controlled by env.PORT

# Run with bun runtime instead of compiled binary
CMD ["bun", "run", "apps/api/src/index.ts"]
```

### Building the API Image

```bash
# Build the image
docker build -f Dockerfile.api -t cossistant-api:latest .

# Run the container
docker run -p 8787:8787 \
  --env-file .env \
  cossistant-api:latest
```

### API Container Features

- **Base Image**: `node:20-slim` for minimal footprint
- **Runtime**: Bun for fast JavaScript/TypeScript execution
- **Build Optimization**: Only builds required dependencies
- **Port**: Controlled by `PORT` environment variable (default: 8787)
- **Monorepo Support**: Uses Turborepo for efficient builds

## Production Deployment

### Complete Docker Compose for Production

Create a `docker-compose.prod.yml`:

```yaml
version: '3.8'

services:
  postgres:
    image: pgvector/pgvector:pg17
    restart: always
    environment:
      POSTGRES_USER: ${DATABASE_USERNAME}
      POSTGRES_PASSWORD: ${DATABASE_PASSWORD}
      POSTGRES_DB: ${DATABASE_NAME}
    volumes:
      - postgres_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U ${DATABASE_USERNAME}"]
      interval: 10s
      timeout: 5s
      retries: 5
    networks:
      - cossistant

  redis:
    image: redis:7-alpine
    restart: always
    volumes:
      - redis_data:/data
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 10s
      timeout: 5s
      retries: 5
    networks:
      - cossistant

  api:
    build:
      context: .
      dockerfile: Dockerfile.api
    restart: always
    ports:
      - "8787:8787"
    env_file:
      - .env
    environment:
      DATABASE_HOST: postgres
      REDIS_URL: redis://redis:6379
    depends_on:
      postgres:
        condition: service_healthy
      redis:
        condition: service_healthy
    networks:
      - cossistant

  workers:
    build:
      context: .
      dockerfile: Dockerfile.workers
    restart: always
    env_file:
      - .env
    environment:
      DATABASE_HOST: postgres
      REDIS_URL: redis://redis:6379
    depends_on:
      postgres:
        condition: service_healthy
      redis:
        condition: service_healthy
    networks:
      - cossistant

volumes:
  postgres_data:
  redis_data:

networks:
  cossistant:
    driver: bridge
```

### Deploy with Production Compose

```bash
# Build and start all services
docker compose -f docker-compose.prod.yml up -d --build

# View logs
docker compose -f docker-compose.prod.yml logs -f

# Scale workers
docker compose -f docker-compose.prod.yml up -d --scale workers=3

# Stop services
docker compose -f docker-compose.prod.yml down
```

## Container Networking

### Service Communication

When running in Docker Compose, services communicate using service names:

- **Database**: `postgres:5432`
- **Redis**: `redis:6379`
- **API**: `api:8787`

Update environment variables:

```bash
DATABASE_HOST=postgres
REDIS_URL=redis://redis:6379
```

### External Access

Expose only the API service to the host:

```yaml
services:
  api:
    ports:
      - "8787:8787"  # Accessible from host
  
  postgres:
    # No ports exposed - only accessible within Docker network
  
  redis:
    # No ports exposed - only accessible within Docker network
```

## Volumes and Data Persistence

### Named Volumes

Docker Compose creates named volumes for data persistence:

- `postgres_data`: PostgreSQL database files
- `redis_data`: Redis RDB snapshots

### Volume Management

```bash
# List volumes
docker volume ls

# Inspect volume
docker volume inspect cossistant_postgres_data

# Backup volume
docker run --rm -v cossistant_postgres_data:/data -v $(pwd):/backup \
  ubuntu tar czf /backup/postgres_backup.tar.gz /data

# Restore volume
docker run --rm -v cossistant_postgres_data:/data -v $(pwd):/backup \
  ubuntu tar xzf /backup/postgres_backup.tar.gz -C /data --strip 1
```

### Bind Mounts for Development

For hot-reloading during development:

```yaml
services:
  api:
    volumes:
      - ./apps/api:/app/apps/api
      - ./packages:/app/packages
```

## Environment Configuration

<Warning>
**Production Security**

- Never include `.env` files in your Docker image
- Use Docker secrets or environment variables for sensitive data
- Generate strong random secrets for production
- Restrict container network access
</Warning>

### Using Environment Files

```bash
# Development
docker compose --env-file .env.dev up

# Production
docker compose --env-file .env.prod up
```

### Docker Secrets (Swarm/Kubernetes)

For orchestrated deployments:

```yaml
secrets:
  db_password:
    external: true
  api_key_secret:
    external: true

services:
  api:
    secrets:
      - db_password
      - api_key_secret
    environment:
      DATABASE_PASSWORD_FILE: /run/secrets/db_password
      API_KEY_SECRET_FILE: /run/secrets/api_key_secret
```

## Troubleshooting

### Container Won't Start

```bash
# Check container logs
docker compose logs api

# Inspect container
docker inspect cossistant-api-1

# Check health status
docker compose ps
```

### Database Connection Issues

```bash
# Test PostgreSQL connection
docker compose exec postgres pg_isready -U postgres

# Connect to database
docker compose exec postgres psql -U postgres -d cossistant

# Check network connectivity
docker compose exec api ping postgres
```

### Redis Connection Issues

```bash
# Test Redis connection
docker compose exec redis redis-cli ping

# Check Redis info
docker compose exec redis redis-cli info
```

### Build Cache Issues

```bash
# Clear build cache
docker builder prune

# Rebuild without cache
docker compose build --no-cache

# Remove all unused data
docker system prune -a --volumes
```

## Performance Optimization

### Multi-stage Builds

Optimize Docker images with multi-stage builds:

```dockerfile
# Build stage
FROM node:20-slim AS builder
WORKDIR /app
COPY . .
RUN bun install && bunx turbo run build

# Production stage
FROM node:20-slim
WORKDIR /app
RUN npm install -g bun
COPY --from=builder /app/dist ./dist
COPY --from=builder /app/node_modules ./node_modules
CMD ["bun", "run", "dist/index.js"]
```

### Layer Caching

Optimize layer caching by copying package files first:

```dockerfile
# Copy package files first (changes less frequently)
COPY package.json bun.lock ./
RUN bun install

# Copy source files (changes more frequently)
COPY . .
RUN bunx turbo run build
```

## Platform Deployment

### Railway

Railway automatically detects and uses Dockerfiles:

1. Connect your GitHub repository
2. Railway detects `Dockerfile.api` and `Dockerfile.workers`
3. Configure environment variables in Railway dashboard
4. Deploy

### Render

Configure `render.yaml`:

```yaml
services:
  - type: web
    name: cossistant-api
    env: docker
    dockerfilePath: ./Dockerfile.api
    envVars:
      - key: DATABASE_HOST
        fromDatabase:
          name: cossistant-db
          property: host

  - type: worker
    name: cossistant-workers
    env: docker
    dockerfilePath: ./Dockerfile.workers
```

### Fly.io

Configure `fly.toml`:

```toml
[build]
  dockerfile = "Dockerfile.api"

[env]
  PORT = "8787"

[[services]]
  http_checks = []
  internal_port = 8787
  protocol = "tcp"
```
