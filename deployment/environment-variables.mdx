## Overview

Cossistant uses environment variables for configuration. All variables are defined in `.env.example` and validated through `apps/api/src/env.ts`.

<Warning>
**Security Notice**

Never commit `.env` files containing real credentials to version control. Always use secure secret management in production.
</Warning>

## Quick Start

```bash
cp .env.example .env
# Edit .env with your configuration
```

## Database Configuration

### DATABASE_HOST

- **Type**: String
- **Required**: Yes
- **Description**: PostgreSQL database host
- **Example**: `localhost`, `postgres.example.com`

### DATABASE_PORT

- **Type**: Number
- **Required**: Yes
- **Default**: `5432`
- **Description**: PostgreSQL port

### DATABASE_USERNAME

- **Type**: String
- **Required**: Yes
- **Description**: PostgreSQL username
- **Example**: `postgres`

### DATABASE_PASSWORD

- **Type**: String
- **Required**: Yes
- **Description**: PostgreSQL password
- **Security**: Use a strong password and store securely

### DATABASE_NAME

- **Type**: String
- **Required**: Yes
- **Description**: PostgreSQL database name
- **Example**: `cossistant`

## Authentication

### BETTER_AUTH_URL

- **Type**: String (URL)
- **Required**: Yes
- **Description**: Base URL for authentication service
- **Example**: `http://localhost:8787` (dev), `https://api.example.com` (prod)
- **Note**: Must match your API deployment URL

### BETTER_AUTH_SECRET

- **Type**: String
- **Required**: Yes
- **Description**: Secret key for signing authentication tokens
- **Security**: Generate a cryptographically random string (32+ characters)
- **Example**: Generate with `openssl rand -base64 32`

### API_KEY_SECRET

- **Type**: String
- **Required**: Yes
- **Description**: Secret for signing and validating API keys
- **Security**: Generate a cryptographically random string (32+ characters)

## Application URLs

### PUBLIC_APP_URL

- **Type**: String (URL)
- **Required**: Yes
- **Description**: Public-facing URL of your frontend application
- **Example**: `http://localhost:3000` (dev), `https://app.example.com` (prod)
- **Used for**: Email links, OAuth callbacks, CORS configuration

### PORT

- **Type**: Number
- **Required**: No
- **Default**: `8787`
- **Description**: Port for the API server to listen on

### QSTASH_CURRENT_SIGNING_KEY

- **Type**: String

## File Storage (S3)

### S3_BUCKET_NAME

- **Type**: String
- **Required**: Yes
- **Description**: S3 bucket name for file storage
- **Example**: `cossistant-uploads`

### S3_REGION

- **Type**: String
- **Required**: Yes
- **Description**: AWS region for S3 bucket
- **Example**: `us-east-1`, `eu-west-1`

### S3_ACCESS_KEY_ID

- **Type**: String
- **Required**: Yes
- **Description**: AWS access key ID with S3 permissions

### S3_SECRET_ACCESS_KEY

- **Type**: String
- **Required**: Yes
- **Description**: AWS secret access key
- **Security**: Keep secure, never commit to version control

### S3_ENDPOINT

- **Type**: String (URL)
- **Required**: No
- **Default**: AWS S3 endpoint
- **Description**: Custom S3-compatible endpoint (MinIO, R2, DigitalOcean Spaces)
- **Example**: `https://s3.us-east-1.amazonaws.com`, `https://minio.example.com`

### S3_FORCE_PATH_STYLE

- **Type**: Boolean
- **Required**: No
- **Default**: `false`
- **Description**: Use path-style URLs instead of virtual-hosted-style
- **Note**: Set to `true` for MinIO or other S3-compatible services

### S3_SIGNED_URL_EXPIRATION_SECONDS

- **Type**: Number
- **Required**: No
- **Default**: `900` (15 minutes)
- **Description**: Expiration time for pre-signed URLs

### S3_PUBLIC_BASE_URL

- **Type**: String (URL)
- **Required**: No
- **Description**: Public base URL for accessing S3 objects directly
- **Example**: `https://cdn.example.com`

### S3_CDN_BASE_URL

- **Type**: String (URL)
- **Required**: No
- **Description**: CDN URL prefix for serving S3 objects
- **Example**: `https://d1234567890.cloudfront.net`

## Redis Cache

### REDIS_URL

- **Type**: String (URL)
- **Required**: Yes
- **Description**: Redis connection URL
- **Example**: `redis://localhost:6379`, `redis://user:pass@redis.example.com:6379`
- **Note**: The `.env.example` shows `http://localhost:8079` but this should be a Redis URL

## Analytics (Tinybird)

### TINYBIRD_HOST

- **Type**: String (URL)
- **Required**: No
- **Default**: `http://localhost:7181`
- **Description**: Tinybird API endpoint
- **Example**: `https://api.tinybird.co` (production)

### TINYBIRD_TOKEN

- **Type**: String
- **Required**: No
- **Default**: `admin` (for local development)
- **Description**: Tinybird API authentication token
- **Setup**: Get from Tinybird dashboard → Auth Tokens

### TINYBIRD_SIGNING_KEY

- **Type**: String
- **Required**: No
- **Description**: JWT signing key (must be workspace ADMIN token)
- **Note**: Falls back to `TINYBIRD_TOKEN` if not set
- **Security**: Use workspace admin token, not personal token

### TINYBIRD_WORKSPACE

- **Type**: String
- **Required**: No
- **Description**: Tinybird workspace identifier

## Environment File Template

Complete `.env.example` reference:

```bash
# Database Configuration
DATABASE_HOST=your_database_host
DATABASE_PORT=5432
DATABASE_USERNAME=your_database_username
DATABASE_PASSWORD=your_database_password
DATABASE_NAME=your_database_name

# Authentication
BETTER_AUTH_URL=http://localhost:8787
BETTER_AUTH_SECRET=your_better_auth_secret

# API Configuration
API_KEY_SECRET=your_api_key_secret
PUBLIC_APP_URL=http://localhost:3000

# OAuth Providers
GOOGLE_CLIENT_ID=your_google_client_id
GOOGLE_CLIENT_SECRET=your_google_client_secret
GITHUB_CLIENT_ID=your_github_client_id
GITHUB_CLIENT_SECRET=your_github_client_secret

# Email Service (Resend)
RESEND_API_KEY=your_resend_api_key
RESEND_AUDIENCE_ID=your_resend_audience_id

# Queue Service (QStash)
QSTASH_URL=https://qstash.upstash.io/v2
QSTASH_TOKEN=your_qstash_token
QSTASH_CURRENT_SIGNING_KEY=your_qstash_current_signing_key
QSTASH_NEXT_SIGNING_KEY=your_qstash_next_signing_key

# File Storage (S3)
S3_BUCKET_NAME=your_s3_bucket_name
S3_REGION=your_s3_region
S3_ACCESS_KEY_ID=your_s3_access_key_id
S3_SECRET_ACCESS_KEY=your_s3_secret_access_key
S3_ENDPOINT=
S3_FORCE_PATH_STYLE=false
S3_SIGNED_URL_EXPIRATION_SECONDS=900
S3_PUBLIC_BASE_URL=
S3_CDN_BASE_URL=

# Redis Cache
REDIS_URL=redis://localhost:6379

# Tinybird Local Analytics
TINYBIRD_HOST=http://localhost:7181
TINYBIRD_TOKEN=
TINYBIRD_SIGNING_KEY=
TINYBIRD_WORKSPACE=

# Public Keys
NEXT_PUBLIC_COSSISTANT_API_KEY=your_public_cossistant_key

# Environment
NODE_ENV=development
```

## Validation

Environment variables are validated at startup through `apps/api/src/env.ts`. Missing required variables will cause the application to fail on startup with clear error messages.
