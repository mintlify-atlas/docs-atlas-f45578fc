The Conversations API allows you to create, retrieve, and manage conversations between visitors and your team. Conversations are the core entity that holds messages, events, and timeline items.

## Authentication

All conversation endpoints support both public and private API keys:

- **Public API Key** (`X-Public-Key` header): For client-side usage, limited to visitor-owned conversations
- **Private API Key** (`Authorization` header): For server-side usage with full access

## Create Conversation

<Card title="POST /conversations" icon="plus">
Create a new conversation with optional initial timeline items (messages).
</Card>

### Endpoint

```
POST https://api.cossistant.com/v1/conversations
```

### Headers

<ParamField header="Authorization" type="string">
  Private API key in Bearer token format: `Bearer sk_test_...`
  
  Use for server-to-server authentication.
</ParamField>

<ParamField header="X-Public-Key" type="string">
  Public API key: `pk_test_...`
  
  Use for browser-based authentication from whitelisted domains.
</ParamField>

<ParamField header="X-Visitor-Id" type="string">
  Visitor ID from localStorage (26-character ULID).
  
  Example: `01JG000000000000000000000`
</ParamField>

### Request Body

<ParamField body="visitorId" type="string">
  Visitor ID. Optional if provided via `X-Visitor-Id` header.
</ParamField>

<ParamField body="conversationId" type="string">
  Optional idempotency key for conversation creation. If provided, it is scoped to one organization+website+visitor tuple. Retries with the same tuple reuse the existing conversation.
</ParamField>

<ParamField body="defaultTimelineItems" type="array" required>
  Array of initial timeline items (messages) to create with the conversation.
  
  Each item contains:
  - `type`: `"message"` or `"event"` (default: `"message"`)
  - `text`: Message text content
  - `parts`: Array of message parts (text, images, files, etc.)
  - `visibility`: `"public"` or `"private"` (default: `"public"`)
  - `visitorId`, `userId`, or `aiAgentId`: Message sender
</ParamField>

<ParamField body="channel" type="string" default="widget">
  Which channel the conversation is from (e.g., `"widget"`, `"email"`, `"api"`)
</ParamField>

### Response

<ResponseField name="conversation" type="object">
  The created conversation object.
  
  <Expandable title="properties">
    <ResponseField name="id" type="string">
      Unique conversation ID (ULID)
    </ResponseField>
    
    <ResponseField name="title" type="string">
      Conversation title (optional)
    </ResponseField>
    
    <ResponseField name="visitorId" type="string">
      Owner visitor ID
    </ResponseField>
    
    <ResponseField name="websiteId" type="string">
      Associated website ID
    </ResponseField>
    
    <ResponseField name="status" type="string">
      Conversation status: `"open"`, `"closed"`, `"resolved"`, or `"spam"`
    </ResponseField>
    
    <ResponseField name="visitorRating" type="number | null">
      Visitor rating (1-5) if provided
    </ResponseField>
    
    <ResponseField name="visitorRatingAt" type="string | null">
      When the rating was submitted
    </ResponseField>
    
    <ResponseField name="createdAt" type="string">
      ISO 8601 timestamp
    </ResponseField>
    
    <ResponseField name="updatedAt" type="string">
      ISO 8601 timestamp
    </ResponseField>
    
    <ResponseField name="lastTimelineItem" type="object">
      The most recent timeline item in the conversation
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="initialTimelineItems" type="array">
  Array of created timeline items
</ResponseField>

### Example Request

```bash cURL
curl -X POST https://api.cossistant.com/v1/conversations \
  -H "X-Public-Key: pk_test_abc123..." \
  -H "X-Visitor-Id: 01JG000000000000000000000" \
  -H "Content-Type: application/json" \
  -d '{
    "conversationId": "unique-conversation-123",
    "defaultTimelineItems": [
      {
        "type": "message",
        "text": "Hello, I need help with my account",
        "visibility": "public",
        "parts": [
          {
            "type": "text",
            "text": "Hello, I need help with my account"
          }
        ]
      }
    ],
    "channel": "widget"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.cossistant.com/v1/conversations', {
  method: 'POST',
  headers: {
    'X-Public-Key': 'pk_test_abc123...',
    'X-Visitor-Id': '01JG000000000000000000000',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    conversationId: 'unique-conversation-123',
    defaultTimelineItems: [
      {
        type: 'message',
        text: 'Hello, I need help with my account',
        visibility: 'public',
        parts: [
          {
            type: 'text',
            text: 'Hello, I need help with my account'
          }
        ]
      }
    ],
    channel: 'widget'
  })
});

const data = await response.json();
```

## List Conversations

<Card title="GET /conversations" icon="list">
Retrieve a paginated list of conversations for a specific visitor.
</Card>

### Endpoint

```
GET https://api.cossistant.com/v1/conversations
```

### Query Parameters

<ParamField query="visitorId" type="string">
  Visitor ID to fetch conversations for. Optional if provided via `X-Visitor-Id` header.
</ParamField>

<ParamField query="page" type="number" default="1">
  Page number for pagination (min: 1)
</ParamField>

<ParamField query="limit" type="number" default="6">
  Number of conversations per page (min: 1, max: 100)
</ParamField>

<ParamField query="status" type="string">
  Filter by conversation status: `"open"` or `"closed"`
</ParamField>

<ParamField query="orderBy" type="string" default="updatedAt">
  Field to order by: `"createdAt"` or `"updatedAt"`
</ParamField>

<ParamField query="order" type="string" default="desc">
  Order direction: `"asc"` or `"desc"`
</ParamField>

### Response

<ResponseField name="conversations" type="array">
  Array of conversation objects
</ResponseField>

<ResponseField name="pagination" type="object">
  Pagination metadata
  
  <Expandable title="properties">
    <ResponseField name="page" type="number">
      Current page number
    </ResponseField>
    
    <ResponseField name="limit" type="number">
      Items per page
    </ResponseField>
    
    <ResponseField name="total" type="number">
      Total number of conversations
    </ResponseField>
    
    <ResponseField name="totalPages" type="number">
      Total number of pages
    </ResponseField>
    
    <ResponseField name="hasMore" type="boolean">
      Whether more pages are available
    </ResponseField>
  </Expandable>
</ResponseField>

### Example Request

```bash cURL
curl -X GET "https://api.cossistant.com/v1/conversations?page=1&limit=10&status=open" \
```

### Request Body

<ParamField body="isTyping" type="boolean" required>
  Whether the visitor is currently typing
</ParamField>

  Optional preview of the visitor's message while typing (max 2000 characters). Only processed when `isTyping` is `true`.

</ParamField>

<ResponseField name="conversationId" type="string">

</ResponseField>

<ResponseField name="isTyping" type="boolean">
  Echo of the reported typing state
</ResponseField>

<ResponseField name="visitorPreview" type="string | null">
  Preview text that was forwarded, or null
</ResponseField>

<ResponseField name="sentAt" type="string">
  ISO 8601 timestamp when the event was recorded
</ResponseField>

### Example Request

```bash cURL

  -H "X-Public-Key: pk_test_abc123..." \
  -H "X-Visitor-Id: 01JG000000000000000000000" \
```
