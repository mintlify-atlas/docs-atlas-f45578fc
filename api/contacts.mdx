---
title: "Contacts"
description: "REST API endpoints for managing contacts and contact organizations"
---

The Contacts API allows you to create, update, and manage contact records for your visitors. Contacts represent identified visitors with additional information like name, email, and custom metadata.

## Authentication

All contact endpoints require a **Public API Key** (`X-Public-Key` header).

## Identify Contact

<Card title="POST /contacts/identify" icon="user-check">
Create or update a contact for a visitor. Links the visitor to the contact record.
</Card>

### Endpoint

```
POST https://api.cossistant.com/v1/contacts/identify
```

### Headers

<ParamField header="X-Public-Key" type="string" required>
  Public API key: `pk_test_...`
</ParamField>

### Request Body

<ParamField body="visitorId" type="string" required>
  The visitor ID to link to the contact (26-character ULID)
</ParamField>

<ParamField body="externalId" type="string">
  External identifier for the contact (e.g., from your CRM or database). Used to find existing contacts.
  
  Either `externalId` or `email` is required.
</ParamField>

<ParamField body="email" type="string">
  The contact's email address. Used to find existing contacts.
  
  Either `externalId` or `email` is required.
</ParamField>

<ParamField body="name" type="string">
  The contact's full name
</ParamField>

<ParamField body="image" type="string">
  URL to the contact's avatar/profile image
</ParamField>

<ParamField body="metadata" type="object">
  Additional custom metadata as key-value pairs. Values can be strings, numbers, booleans, or null.
  
  Example: `{ "plan": "premium", "role": "admin", "signupDate": "2024-01-01" }`
</ParamField>

<ParamField body="contactOrganizationId" type="string">
  The contact organization ID this contact belongs to
</ParamField>

### Response

<ResponseField name="contact" type="object">
  The created or updated contact
  
  <Expandable title="properties">
    <ResponseField name="id" type="string">
      Unique contact ID (ULID)
    </ResponseField>
    
    <ResponseField name="externalId" type="string | null">
      External identifier
    </ResponseField>
    
    <ResponseField name="name" type="string | null">
      Contact's name
    </ResponseField>
    
    <ResponseField name="email" type="string | null">
      Contact's email
    </ResponseField>
    
    <ResponseField name="image" type="string | null">
      Avatar/profile image URL
    </ResponseField>
    
    <ResponseField name="metadata" type="object | null">
      Custom metadata
    </ResponseField>
    
    <ResponseField name="contactOrganizationId" type="string | null">
      Associated organization ID
    </ResponseField>
    
    <ResponseField name="websiteId" type="string">
      Website ID
    </ResponseField>
    
    <ResponseField name="organizationId" type="string">
      Organization ID
    </ResponseField>
    
    <ResponseField name="userId" type="string | null">
      Linked user ID if applicable
    </ResponseField>
    
    <ResponseField name="createdAt" type="string">
      ISO 8601 timestamp
    </ResponseField>
    
    <ResponseField name="updatedAt" type="string">
      ISO 8601 timestamp
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="visitorId" type="string">
  The visitor ID that was linked to the contact
</ResponseField>

### Example Request

```bash cURL
curl -X POST https://api.cossistant.com/v1/contacts/identify \
  -H "X-Public-Key: pk_test_abc123..." \
  -H "Content-Type: application/json" \
  -d '{
    "visitorId": "01JG000000000000000000000",
    "externalId": "user_12345",
    "email": "john.doe@example.com",
    "name": "John Doe",
    "image": "https://example.com/avatar.png",
    "metadata": {
      "plan": "premium",
      "role": "admin",
      "company": "Acme Corp"
    }
  }'
```

```javascript JavaScript

```

## Create Contact

<Card title="POST /contacts" icon="user-plus">
Create a new contact. If `externalId` is provided and already exists, the contact is updated.
</Card>

### Endpoint

```
POST https://api.cossistant.com/v1/contacts
```

### Request Body

<ParamField body="externalId" type="string">
  External identifier for the contact. If provided and exists, performs an upsert.
</ParamField>

<ParamField body="name" type="string">
  The contact's name
</ParamField>

<ParamField body="email" type="string">
  The contact's email address
</ParamField>

<ParamField body="image" type="string">
  URL to the contact's avatar
</ParamField>

<ParamField body="metadata" type="object">
  Custom metadata as key-value pairs
</ParamField>

<ParamField body="contactOrganizationId" type="string">
  Contact organization ID
</ParamField>

### Response

Returns the created or updated contact object (same structure as Identify Contact).

### Example Request

```bash cURL
curl -X POST https://api.cossistant.com/v1/contacts \
  -H "X-Public-Key: pk_test_abc123..." \
  -H "Content-Type: application/json" \
  -d '{
    "externalId": "user_67890",
    "name": "Jane Smith",
    "email": "jane.smith@example.com",
    "metadata": {
      "plan": "enterprise",
      "signupDate": "2024-01-10"
    }
  }'
```

### Example Response

```json 201 - Created

Update an existing contact.
</Card>

### Endpoint

```
PATCH https://api.cossistant.com/v1/contacts/{id}
```

### Path Parameters

<ParamField path="id" type="string" required>
  The contact ID (ULID)
</ParamField>

### Request Body

All fields are optional. Only provided fields will be updated.

<ParamField body="externalId" type="string">

</ParamField>

<ParamField body="name" type="string">

</ParamField>

<ParamField body="email" type="string">

</ParamField>

```

</ParamField>

Retrieve a contact organization by ID.
