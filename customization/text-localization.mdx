The Cossistant support widget includes a fully typed translation system with built-in support for multiple languages and dynamic text formatting.

## Built-in Locales

The widget ships with three complete translations:

- **English** (`en`) - Default
- **Spanish** (`es`)
- **French** (`fr`)

```tsx
import { Support } from '@cossistant/react';

// Use Spanish
<Support locale="es" />

// Use French
<Support locale="fr" />
```

## Overriding Text

Customize any text using the `content` prop:

<Tabs>
  <Tab title="Simple String">
    ```tsx
    <Support
      content={{
        "common.actions.askQuestion": "Contact us",
        "page.home.tagline": "Get help 24/7"
      }}
    />
    ```
  </Tab>
  <Tab title="Per-Locale">
    ```tsx
    <Support
      locale="es"
      content={{
        "common.actions.askQuestion": {
          en: "Contact us",
          es: "Contáctanos",
          fr: "Contactez-nous"
        }
      }}
    />
    ```
  </Tab>
  <Tab title="Dynamic Function">
    ```tsx
    <Support
      content={{
        "page.home.greeting": ({ variables, context, utils }) => {
          const period = utils.timeOfDay();
          const name = variables?.visitorName ?? context.visitor?.contact?.name;
          
          if (period.token === "evening") {
            return `Good evening${name ? ` ${name}` : ""} 👋`;
          }
          
          return `Hey${name ? ` ${name}` : ""}, how can we help?`;
        }
      }}
    />
    ```
  </Tab>
</Tabs>

## Using Text in Custom Components

Access translations in your custom pages and components:

<Tabs>
  <Tab title="Text Component">
    ```tsx
    import { Text } from '@cossistant/react/support';
    
    function CustomPage() {
      return (
        <div>
          <h1>
            <Text textKey="page.home.greeting" />
          </h1>
          <button>
            <Text textKey="common.actions.askQuestion" />
          </button>
        </div>
      );
    }
    ```
  </Tab>
  <Tab title="useSupportText Hook">
    ```tsx
    import { useSupportText } from '@cossistant/react/support';
    
    function CustomButton() {
      const text = useSupportText();
      
      return (
        <button title={text("common.actions.askQuestion")}>
          {text("page.home.history.more", { count: 5 })}
        </button>
      );
    }
    ```
  </Tab>
</Tabs>

## Translation Functions

Keys can be functions with access to context and utilities:

```tsx
<Support
  content={{
    "page.home.greeting": ({ variables, context, utils }) => {
      // Available utilities
      const timeOfDay = utils.timeOfDay(); // { token: 'morning' | 'afternoon' | 'evening', label: string }
      const formatted = utils.formatNumber(1234); // "1,234"
      const plural = utils.pluralize(5, { one: "item", other: "items" }); // "items"
      const title = utils.titleCase("hello world"); // "Hello World"
      
      // Available context
      const websiteName = context.website?.name;
      const visitorName = context.visitor?.contact?.name;
      const humanAgents = context.humanAgents; // Array of available agents
      const aiAgents = context.aiAgents; // Array of AI agents
      
      // Variables passed to this specific key
      const name = variables?.visitorName;
      
      return `Hello ${name || visitorName}!`;
    }
  }}
/>
```

### Available Utilities

<Tabs>
  <Tab title="formatNumber">
    Format numbers with locale-aware formatting:
    
    ```tsx
    utils.formatNumber(1234); // "1,234" (en)
    utils.formatNumber(1234.56, { maximumFractionDigits: 2 }); // "1,234.56"
    ```
  </Tab>
  <Tab title="pluralize">
    Select singular or plural form based on count:
    
    ```tsx
    utils.pluralize(1, { one: "message", other: "messages" }); // "message"
    utils.pluralize(5, { one: "message", other: "messages" }); // "messages"
    ```
  </Tab>
  <Tab title="titleCase">
    Convert text to title case:
    
    ```tsx
    utils.titleCase("hello world"); // "Hello World"
    utils.titleCase("SUPPORT TEAM"); // "Support Team"
    ```
  </Tab>
  <Tab title="timeOfDay">
    Get current time period:
    
    ```tsx
    const period = utils.timeOfDay();
    // { token: 'morning' | 'afternoon' | 'evening', label: 'Morning' }
    
    const greeting = {
      morning: "Good morning",
      afternoon: "Good afternoon",
      evening: "Good evening"
    }[period.token];
    ```
  </Tab>
</Tabs>

## Adding a New Language

Add support for additional locales:

<Tabs>
  <Tab title="1. Declare Locale">
    Extend the locale type in your app:
    
    ```typescript
    // types/cossistant.d.ts
    import '@cossistant/react/support';
    
    declare module '@cossistant/react/support' {
      export type SupportLocale = 'en' | 'es' | 'fr' | 'de' | 'ja';
    }
    ```
  </Tab>
  <Tab title="2. Provide Translations">
    Supply translations for all required keys:
    
    ```tsx
    // translations/de.ts
    export const germanTranslations = {
      "common.actions.askQuestion": "Stellen Sie uns eine Frage",
      "common.actions.attachFiles": "Dateien anhängen",
      "page.home.greeting": ({ variables, utils }) => {
        const period = utils.timeOfDay();
        const greetings = {
          morning: "Guten Morgen",
          afternoon: "Guten Tag",
          evening: "Guten Abend"
        };
        return greetings[period.token];
      },
      // ... all other keys
    };
    ```
  </Tab>
  <Tab title="3. Use Custom Locale">
    ```tsx
    import { germanTranslations } from './translations/de';
    
    <Support
      locale="de"
      content={germanTranslations}
    />
    ```
  </Tab>
</Tabs>


## Type Safety

The translation system is fully typed:

```tsx
// ✅ Valid - key exists
text("common.actions.askQuestion");

// ✅ Valid - with required variables
text("page.home.history.more", { count: 5 });

// ❌ TypeScript error - key doesn't exist
text("invalid.key");

// ❌ TypeScript error - missing required variable
text("page.home.history.more");
```

## Best Practices

<Accordion title="Use functions for dynamic content">
  When text needs to change based on context, use function-based translations:
  
  ```tsx
  "page.home.greeting": ({ context, utils }) => {
    const hour = new Date().getHours();
    const name = context.visitor?.contact?.name;
    
    if (hour < 12) return `Good morning${name ? ` ${name}` : ""}`;
    if (hour < 18) return `Good afternoon${name ? ` ${name}` : ""}`;
    return `Good evening${name ? ` ${name}` : ""}`;
  }
  ```
</Accordion>

<Accordion title="Keep fallbacks consistent">
  Ensure fallback text makes sense in all contexts where it might appear.
</Accordion>

<Accordion title="Test all locales">
  When overriding keys, test with different locales to ensure text displays correctly.
</Accordion>
