## What are Conversations?

A **conversation** is a threaded chat session between a [visitor](/concepts/contacts) (or [contact](/concepts/contacts)) and your support team. Each conversation contains a timeline of [timeline items](/concepts/timeline-items)—messages, events, tool invocations, and system activities—that provide a complete history of the interaction.

Conversations are the central organizing unit in Cossistant, bringing together visitors, agents (human and AI), and contextual data into a single, real-time thread.

## Why Conversations Matter

Conversations provide:

- **Unified context**: All messages, metadata, and actions in one place
- **Real-time collaboration**: Human and AI agents working together seamlessly
- **Complete audit trail**: Every action, decision, and state change is tracked
- **Cross-device continuity**: For identified contacts, conversations persist across devices
- **Smart routing**: Automatic assignment, escalation, and categorization

## Conversation Lifecycle

### 1. Creation

Conversations are created when:

- A visitor sends their first message through the widget
- A support agent initiates outreach via the dashboard
- Your backend creates one via the REST API

```typescript
// Create a conversation via API
const response = await fetch('/api/conversations', {
  method: 'POST',
  headers: {
    'X-Website-Id': websiteId,
    'X-Visitor-Id': visitorId,
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    defaultTimelineItems: [
      {
        type: 'message',
        text: 'Hello, I need help with my account',
        parts: [
          { type: 'text', text: 'Hello, I need help with my account' }
        ],
      }
    ],
    channel: 'widget',
  }),
});
```

### 2. Status Flow

Conversations move through these states:

```
open → resolved → spam
  ↓       ↑
  └───────┘ (can be reopened)
```

- **open**: Active conversation requiring attention
- **resolved**: Conversation marked as complete (can be reopened)
- **spam**: Blocked conversation from abusive visitor

### 3. Priority Management

Conversations can be prioritized to help teams focus on urgent issues:

- **low**: General questions, non-urgent feedback
- **normal**: Standard support requests (default)
- **high**: Important issues affecting user experience
- **urgent**: Critical problems requiring immediate attention

<Note>
AI agents can automatically adjust priority based on conversation content and sentiment analysis.
</Note>

## Real-time Features

### WebSocket Synchronization

Conversations support real-time updates via WebSocket:

- New messages appear instantly for all participants
- Typing indicators show when agents are composing
- Seen receipts track when messages are read
- Status changes broadcast immediately to all connected clients

The Cossistant SDK handles all WebSocket management automatically.

### Typing Indicators

The system tracks typing state for both visitors and agents:

```typescript
// Set typing state for a visitor
await fetch(`/api/conversations/${conversationId}/typing`, {
  method: 'POST',
  body: JSON.stringify({
    isTyping: true,
    visitorPreview: 'I am having trouble with...',
  }),
});
```

### Seen Tracking

Cossistant automatically tracks when participants last viewed a conversation:


## Multi-Agent Collaboration

Conversations support multiple types of participants working together:

### Participant Types

- **Visitors**: The end users initiating conversations
- **Human agents**: Support team members from your organization
- **AI agents**: Automated assistants that can respond, escalate, and manage conversations

### AI Agent Escalation

When an AI agent determines it cannot resolve an issue, it can escalate to human support:

```typescript
// Escalation is tracked on the conversation
conversation.escalatedAt; // When escalation occurred
conversation.escalatedByAiAgentId; // Which AI agent escalated
conversation.escalationReason; // Why it was escalated
conversation.escalationHandledAt; // When a human took over
conversation.escalationHandledByUserId; // Which human agent handled it
```

<Info>
AI agents can be configured to automatically assign conversations to specific team members when escalating.
</Info>

## Real-World Example

Here's how a conversation flows through the system:

```typescript
// 1. Visitor starts conversation
const conv = await createConversation({
  visitorId: "vis_abc123",
  websiteId: "web_xyz789",
  defaultTimelineItems: [
    {
      type: "message",
      text: "I can't access my account",
      parts: [{ type: "text", text: "I can't access my account" }],
    }
  ],
});
// Status: open, Priority: normal

// 2. AI agent responds and analyzes sentiment
// Sentiment: negative, Confidence: 0.85
// Priority automatically raised to: high

// 3. AI agent realizes this requires password reset (account access)
// Escalates to human support team
conversation.escalatedAt = "2024-03-13T10:15:00Z";
conversation.escalationReason = "Account access issue requires manual verification";

// 4. Human agent Sarah joins
// conversationParticipant created for Sarah
// conversationAssignee created: assignedByAiAgentId = aiAgent.id

// 5. Sarah resolves the issue
conversation.status = "resolved";
conversation.resolvedByUserId = "user_sarah";
conversation.resolvedAt = "2024-03-13T10:30:00Z";
conversation.resolutionTime = 900; // 15 minutes

// 6. Visitor rates the experience
conversation.visitorRating = 5;
conversation.visitorRatingAt = "2024-03-13T10:31:00Z";
```

## Cross-Device Continuity

For identified [contacts](/concepts/contacts), conversations sync across devices:

- Start conversation on desktop → Continue on mobile
- Same history, same context, seamless experience

Anonymous [visitors](/concepts/contacts) have device-specific conversations tied to their visitor ID.

## Best Practices

### Auto-Generate Titles

Enable AI-generated conversation titles for better organization:

```typescript
aiAgent.behaviorSettings.autoGenerateTitle = true;
```

### Configure Escalation

Set up default escalation routing:

```typescript
aiAgent.behaviorSettings.canEscalate = true;
aiAgent.behaviorSettings.defaultEscalationUserId = "user_lead_agent";
aiAgent.behaviorSettings.autoAssignOnEscalation = true;
```

### Monitor Sentiment

Use sentiment analysis to identify urgent issues:

```typescript
aiAgent.behaviorSettings.autoAnalyzeSentiment = true;

// Then filter for negative sentiment conversations
const urgentConversations = conversations.filter(
  c => c.sentiment === "negative" && c.sentimentConfidence > 0.7
);
```
